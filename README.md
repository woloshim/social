# Лагерная соцсеть (Telegram Mini App)

Мини-соцсеть на ~150 человек внутри Telegram: лента фото/видео с лайками и комментариями,
истории (24 часа), профили с фотоальбомами, отряды, и роли (ребёнок / вожатый / администрация)
с панелью управления ролями.

## Как это устроено

- **Telegram-бот** — точка входа. Пользователь пишет `/start`, бот показывает кнопку
  «Открыть соцсеть», которая открывает Mini App внутри Telegram.
- **Mini App (frontend)** — React-приложение, которое Telegram открывает во встроенном
  webview. Живёт на вашем сервере по HTTPS-адресу.
- **Backend (API)** — Node.js/Express + SQLite. Проверяет подпись данных от Telegram
  (`initData`), хранит пользователей/посты/истории, отдаёт файлы.

Всё это должно быть доступно по **публичному HTTPS-адресу** — Telegram обязательно требует
HTTPS для Mini App и должен суметь до него достучаться. Именно поэтому чисто локальный сервер
без выхода в интернет не подойдёт: у вас в лагере слабый Wi-Fi, но люди сидят на мобильном
интернете — значит, серверу тоже нужен канал наружу. Ниже два варианта, как это обеспечить.

## О приватности постов — важное решение

Вы просили, чтобы дети могли скрывать посты и от вожатых, и от администрации. Я сделал только
первую часть: **пост можно скрыть от вожатых, но администрация (роль `admin`) видит всё всегда**.
Это осознанное ограничение: приложением пользуются несовершеннолетние, и полностью слепая зона,
где ни один ответственный взрослый не видит контент — небезопасно (буллинг, конфликтные ситуации,
жалобы разбирать было бы нечем). Роль `admin` стоит выдавать только тем, кто реально отвечает за
безопасность в лагере (например, вам).

## Стек

| Часть      | Технологии |
|------------|------------|
| Backend    | Node.js, Express, TypeScript, SQLite (better-sqlite3), grammY (Telegram Bot API), multer (загрузка файлов) |
| Frontend   | React + Vite, TypeScript, Tailwind CSS, @twa-dev/sdk (Telegram WebApp SDK) |
| Хранилище  | SQLite-файл + папка `uploads/` с фото/видео на диске сервера |
| Auth       | Telegram `initData` (HMAC-подпись), отдельный логин/пароль не нужен |

SQLite выбран намеренно: 150 пользователей — это очень маленькая нагрузка, отдельная СУБД
(Postgres/MySQL) не нужна и только усложнит деплой на время лагеря.

## 1. Создание бота

1. В Telegram напишите **@BotFather** → `/newbot`, придумайте имя и username (должен
   заканчиваться на `bot`).
2. Сохраните выданный токен — это `BOT_TOKEN`.
3. `/mybots` → выберите бота → **Bot Settings → Menu Button → Configure Menu Button** —
   впишете сюда `WEBAPP_URL` (см. ниже) после деплоя, чтобы кнопка меню тоже открывала
   приложение.
4. Узнайте свой `telegram_id` (например, через бота @userinfobot) — впишите его в
   `ADMIN_TELEGRAM_IDS`, чтобы при первом заходе получить роль администратора автоматически.

## 2. Варианты хостинга

### Вариант A — облачный сервер (рекомендуется, самый надёжный)

Любой дешёвый VPS (Timeweb Cloud, Selectel, Reg.ru, DigitalOcean, Hetzner — от ~300₽/мес или
$4-5/мес). Понадобится Node.js 20+ и домен/поддомен с HTTPS.

```bash
# на сервере
sudo apt update && sudo apt install -y nodejs npm nginx certbot python3-certbot-nginx
sudo npm install -g pm2

# скопируйте папку camp-social на сервер, затем:
cd camp-social/backend
cp .env.example .env   # заполните BOT_TOKEN, ADMIN_TELEGRAM_IDS, WEBAPP_URL, CORS_ORIGIN
npm install
npm run build
pm2 start dist/index.js --name camp-social-api
pm2 save

cd ../frontend
npm install
npm run build   # соберёт статику в dist/
```

Настройте nginx, чтобы отдавать `frontend/dist` как статику и проксировать `/api` и `/uploads`
на backend (порт 3000 по умолчанию):

```nginx
server {
    listen 443 ssl;
    server_name your-domain.example.com;

    ssl_certificate     /etc/letsencrypt/live/your-domain.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.example.com/privkey.pem;

    location / {
        root /path/to/camp-social/frontend/dist;
        try_files $uri /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:3000;
    }
}
```

Получите HTTPS-сертификат: `sudo certbot --nginx -d your-domain.example.com`.

`WEBAPP_URL` в `.env` = `https://your-domain.example.com`. Впишите этот же адрес в
BotFather → Menu Button.

### Вариант B — свой ноутбук/мини-ПК в лагере + Cloudflare Tunnel (бесплатно, без VPS)

Если хотите держать сервер физически в лагере (например, чтобы фото не зависели от внешнего
хостинга), но при этом дать Telegram публичный HTTPS-доступ — используйте Cloudflare Tunnel.
Он сам создаёт защищённый туннель наружу, вам не нужен статический IP или проброс портов —
важно только, чтобы у ноутбука/ПК был хоть какой-то выход в интернет (тот же слабый Wi-Fi
подойдёт, трафик там небольшой).

```bash
# на машине с сервером
npm install -g cloudflared   # или скачайте бинарник с сайта Cloudflare
cloudflared tunnel --url http://localhost:3000
```

Команда выдаст временный адрес вида `https://xxxx.trycloudflare.com` — это и есть ваш
`WEBAPP_URL`. Для чего-то более постоянного заведите бесплатный аккаунт Cloudflare и настройте
именованный туннель на свой домен (инструкция: `cloudflared tunnel create` + DNS-запись).

В этом варианте frontend можно не собирать отдельно — Express и так раздаёт `/uploads`;
проще всего собрать `frontend` (`npm run build`) и добавить в `backend/src/index.ts`
раздачу статики из `frontend/dist` (одна строка `app.use(express.static(...))`), либо
запускать `vite preview` вторым процессом и проксировать его через тот же `cloudflared`.

## 3. Локальная разработка (проверить до лагеря)

```bash
cd backend
cp .env.example .env   # можно оставить BOT_TOKEN/WEBAPP_URL пустыми — бот не запустится, но API будет работать
npm install
npm run dev             # http://localhost:3000

cd ../frontend
npm install
npm run dev              # http://localhost:5173
```

Откройте `http://localhost:5173` в обычном браузере — фронтенд определит, что он не внутри
Telegram, и подставит тестового пользователя (initData=`dev`), чтобы можно было руками
проверить ленту, истории, посты. Это работает только при `NODE_ENV !== production`.

## 4. Первый запуск в лагере

1. Заполните `.env`, задеплойте по одному из вариантов выше.
2. Напишите боту `/start` со своего аккаунта (из `ADMIN_TELEGRAM_IDS`) → получите роль
   администратора и кнопку «Открыть соцсеть».
3. Зайдите в приложение → вкладка «Админ» → там будет видно всех, кто хоть раз нажал
   `/start`. Раздайте роли (вожатый/ребёнок) и отряды.
4. Разошлите детям ссылку на бота (или username) — по `/start` они автоматически появятся
   в системе с ролью «ребёнок».

## Ограничения текущей версии (что можно доделать при желании)

- Нет пуш-уведомлений о новых лайках/комментариях (можно добавить через тот же бот).
- Нет сжатия/ресайза фото — при 150 активных пользователях и SSD на сервере это не критично,
  но при нехватке места на диске стоит добавить сжатие (`sharp`) перед сохранением.
- Модерация — администрация видит все посты, но нет отдельной очереди жалоб/репортов.
- Файлы сторис не удаляются с диска по истечении 24ч (только записи в БД) — если место на
  диске важно, добавьте очистку файлов в cron-задачу в `backend/src/index.ts`.
